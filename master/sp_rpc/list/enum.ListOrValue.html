Expand description

RPC list or value wrapper.

For some RPCs itโs convenient to call them with either a single value or a whole list of values to get a proper response. In theory you could do a batch query, but itโs:

  1. Less convenient in client libraries
  2. If the response value is small, the protocol overhead might be dominant.

Also itโs nice to be able to maintain backward compatibility for methods that were initially taking a value and now we want to expand them to take a list.
